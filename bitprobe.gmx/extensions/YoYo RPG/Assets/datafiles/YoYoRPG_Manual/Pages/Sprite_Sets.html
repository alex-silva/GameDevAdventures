Sprite Sets

In this section we will discuss the sprites that the YoYo Games RPG engine requires and how the scripts that set them up for each instance are structured. These sprite sets are only applicable to objects that move and can be interacted with, for example NPCs, the player, or enemies. Note that the variables used in these scripts provide an easy way to change the graphics used to display an instance, but they do not affect the instance in any other way. This means that you could, if you wished, create an NPC that uses the "Scorpion" sprite set and so have a talking scorpion in your game!

Required Sprites

The YoYo Games RPG engine is a top down engine and requires four sprites (one for each direction) for each of the three mobile character "states" (making a total of 12 directional sprites) as well as a couple of extra sprites for the death particles and the character avatar (used in Dialogue). These sprites and states correspond to the following:

StateSprites
WalkingNorth, South, East, West
FightingNorth, South, East, West
Staggering (when hit)North, South, East, West
DeathSingle sprite with multiple "bits" as sub-images
AvatarSingle sprite of the "head" of the character

These sprites will be assigned to a number of variables in the instance being created which will then be used in further scripts to correctly get your characters facing the required direction and have the correct behaviour set for the circumstances.

The Scripts

If you look in the "Scripts" folder of the resource tree, you will see a group of scripts titled "Sprite Sets". These scripts are where each instance or object will set up it's sprite properties. If you open up the "objTestNPC" (from the Mobiles>NPCs object folders), you can see that we call two scripts from the Create Event:

SetAvatar("J. Doe", -1);
SetSpritesNPCMale();


The first script sets some avatar information and has the following arguments:

ArgumentDescription
name The name of the NPC or character (use an empty string "" for no name).
avatar_sprite The avatar sprite to use (-1 is no avatar).
Note that the avatar sprite is also set in the SetSprites* script, so you can use this to overwrite the base avatar with another. Using our "NPC as a scorpion" example for before, you could use the SetAvatar script after the SetSprite* script to give the NPC a human avatar, indicating that even though you are talking to a scorpion, it's really a human!

The second script will be unique to each of the chracters that you wish to animate, and sets variables for each of the directions and death particles. So, if you make a new character you should duplicate one of the other SetSprite* scripts and simply fill in the sprite indices required.

Note that in the demo game these scripts are often called from the Creation Code for an instance of an object (like NPCs) to give distinct sprites while keeping the defined object behaviour.

Before continuing, you should open some of these sprite set scripts now and see how they are structured.
